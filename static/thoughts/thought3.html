MY COLLECTION OF GOOD UX PRACTICES

Beautiful design comes last, functionality first, duh

There is a trend of prettification at the expense of functionality (Yahoo Finance). People. however, do not use products because they are nice, people use them because they solve problems. Always do the ground work first, Information architecture, interactions design, wire framing, and only then should you reinforce your design with a beautiful visual representation.

Preserve state

Let’s consider a situation when you are doing something on Screen1, perhaps you are searching for a house to rent on a large map that occupies the whole browser window, you find an interesting property and click on it to get more info, the next thing you know is that you now are on a totally different Screen2 with property’s info and you are completely clueless about what have happened to the map and to those filters that you have been adjusting for 10 minutes. Most likely, if you hit a “back” button you will have to re-adjust them once again. Never do this, open a new tab, show a pop-up or expand a section inside Screen1, but never change state without clearly letting a user know what is about to happen.

Be careful with pagination

Scrolling is faster then pagination, unless there is a particular need to have pagination, consider going with scrolling. Inappropriately used pagination can be very frustrating, especially for content-centric products with a lot of content.

Don’t overdo animations

Animations are beautiful, but when they are overused, it can be frustrating. Animations must not take too long to complete and must serve a clear purpose. If one of the points above is violated, it is probably better to change it or not to use an animation at all.

Use familiar UI elements where possible

It is almost always bad if you have to educate a user (it is not, only in cases when you want to create an artificial barrier to filter some users out e.g. Snapchat or to prevent existing users from switching to a competitor e.g. IOS & Android). By using UI elements that user is already familiar with you significantly improve their overall experience by eliminating a need to invest time into learning how to use your product. If you want to stand out, there are other ways to do so.

Avoid mystery

Minimalist design is good, but it is not a reason not to tell your users what a button does or where a link leads. Always make it clear what each element does, don’t make users guess, or they will just avoid interacting with an element altogether.

Make sure that user is not trapped

Sometimes designers, in pursuit of simplicity, do not allow a user to leave a screen without making a change (often seen in filter menus) or to get back from where he/she came. This is not such a big issue for websites ( still not good though ) because a user can always push a back button, in apps, however, such a situation can be very frustrating. Always clearly show a user where he is in a process and leave a way to come back, or to safely discard changes.

Carefully select what information to provide a user with

Doing a good job on information architecture part can seem obvious, but there are too many cases of poor information architecture out there. Brutally cut unnecessary stuff, and make extra sure that a user can always get where he needs and then go back (hello access management in JIRA). What is unnecessary stuff? Anything that has a potential to sway a user away from a way you want him/her to follow.

Develop an element convention

Element convention is another old but often overlooked practice. To make it work right, develop element classes e.g. buttons, links, etc. and make each instance of a class inherit a visual and perhaps an interaction style. It will instantly make using your product more intuitive.

Use icons with text labels where possible

Ambiguity is an enemy.

Less words

Use infographics, charts, images where possible, they are always much better in conveying information than text. In fact, users almost never read large text pieces unless they are really motivated.
